# 日报项目 · 周报回归测试

## 背景

现有日报简报系统（ai-briefing）已稳定运行：采集→LLM润色→HTML→邮件推送。但缺乏一个自动化回归测试来验证周报全链路是否正常。

## 需求

一个回归测试脚本，模拟周报的完整流程并验证每个环节的输出。

### 测试流程

1. **采集环节** — 运行 `collect_weekly()`，验证返回的数组不为空
2. **HTML 生成** — 读取 HTML 模板，验证模板占位符能否被正确替换
3. **邮件发送** — 发送一封测试邮件到用户的邮箱
4. **链路验证** — 用户确认收到邮件即证明全链路通

### 验证条件

- 采集数据应为有效项目列表（每个项目有 name/url/description）
- HTML 模板渲染后不应残留 `##PLACEHOLDER##` 占位符
- 邮件发送不报错
- 用户最终邮箱收到测试邮件

### 约束

- 使用现有代码，不重复实现采集/HTML/邮件发送逻辑
- 测试邮件主题应有 `[回归测试]` 前缀，方便区分
- 不影响现有的 cron 定时任务（不发真实简报）
- 测试完成后输出一份清晰的测试报告

### 验收标准

1. 运行回归测试脚本后，屏幕上输出各环节的测试结果（PASS/FAIL）
2. 用户邮箱（sunzcdev@gmail.com）收到一封带 `[回归测试]` 前缀的邮件
3. 所有环节通过后脚本返回 exit code 0
